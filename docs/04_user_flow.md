# User Flow

# Echo

## Overview

This document outlines the journey a user takes while interacting with Echo, from signing up to building a connected knowledge network.

The objective is to create an experience that feels calm, intuitive, and encourages connected thinking rather than simple note storage.

---

# Primary User Flow

## 1. Welcome

User opens Echo.

↓

Landing Page

↓

Click **Get Started**

↓

Register / Login

↓

Dashboard

---

## 2. Dashboard

The dashboard welcomes the user with:

* Greeting
* Recent Ideas
* Favorites
* Reflection Prompt
* Search Bar
* Create Idea Button

↓

User chooses an action.

---

## 3. Create an Idea

Click **+ New Idea**

↓

Enter:

* Title
* Description
* Tags

↓

Save

↓

Idea appears in Dashboard

---

## 4. Connect Ideas

Open an Idea

↓

Click **Connect Idea**

↓

Search Existing Ideas

↓

Select One

↓

Relationship Created

↓

Both ideas now reference each other.

---

## 5. Search

User enters keyword

↓

Echo searches:

* Titles
* Descriptions
* Tags

↓

Relevant ideas displayed.

---

## 6. Journal

Click Journal

↓

Create Daily Reflection

↓

Save

↓

Journal stored by date.

---

## 7. Favorites

Open any idea

↓

Click ⭐ Favorite

↓

Idea appears in Favorites section.

---

## 8. Logout

Profile

↓

Logout

↓

Return to Login Screen

---

# Error Flow

Invalid Login

↓

Display Error Message

↓

Retry Login

---

Search Returns No Results

↓

Display

"No matching ideas found."

↓

Offer to create a new idea.

---

Internet Connection Lost

↓

Display Offline Message

↓

Retry Connection

---

# Design Principles

Echo should always feel:

* Calm
* Minimal
* Fast
* Thoughtful
* Easy to navigate

Users should never need more than three clicks to perform common tasks.
